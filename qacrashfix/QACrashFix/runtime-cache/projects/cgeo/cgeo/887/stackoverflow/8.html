i get this error every time i want to display image from web using build in gallery in Android

First, there is no single "build in gallery" application. There are hundreds upon hundreds of devices, and they will have a variety of "build in gallery" applications.

Second, there is nothing on your device capable of "display image from web", other than the browser, and then only if you let Android determine the actual concrete MIME type. There is no requirement that any device's "build in gallery" application support third-party apps using it to download and display images.

@pawelo: "I need autorotate and multitouch resizing" -- then code that yourself. First, ACTION_VIEW is to view an image, not to edit an image. Second, there is no guarantee that a user will have an app that offers "autorotate and multitouch resizing" that you can ask to edit your image for you. If you wish to more generically allow the user to download and edit an image, download the image yourself, the use an ACTION_EDIT Intent with the correct MIME type... but this would allow the user to do any sort of editing, and only if they have an image editor installed. –  CommonsWare Aug 3 '12 at 11:23
